## How it works

Syfrah is a **CLI orchestrator** — not a daemon. It configures system services (WireGuard, systemd), then exits. The kernel does the heavy lifting.

```bash
# Server 1: create a mesh
syfrah hypervisor init --region eu --zone fsn1 --peering

# Server 2: join the mesh
syfrah hypervisor join --target 46.224.166.60 --pin G7CCZX --region eu --zone nbg1

# Check status
syfrah hypervisor status
```

## Design principles

1. **Hypervisor-centric** — every server is a hypervisor. The mesh connects them.
2. **No daemon** — syfrah configures systemd services and exits. WireGuard runs in the kernel.
3. **Declarative resources** — a single `ResourceDef` generates both CLI commands and REST API routes.
4. **IPv6-native** — each mesh gets a ULA /48 prefix, each node a /128 address.
5. **Manual peering** — no automatic discovery. Operators approve every join via PIN or interactive prompt.
